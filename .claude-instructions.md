# ZipPicks Production Build Instructions

## Project Context
Building enterprise-ready ZipPicks features directly into production folder.
After generation, code will be uploaded to live server.

## Build Requirements
Every feature must be SERVER-READY with:
1. **Zero dependencies on local environment**
2. **Production-ready error handling**
3. **Server-optimized file structure**
4. **Immediate deployment capability**

## File Generation Standards

### Always Generate These Files:
- `server.js` - Main production server
- `package.json` - All required dependencies
- `README.md` - Deployment instructions
- `.env.example` - Environment variables
- `startup.sh` - Server startup script

### Code Must Be:
- **Self-contained** - No external file dependencies
- **Server-optimized** - Fast startup, low memory usage
- **Error-resilient** - Graceful failure handling
- **Monitoring-ready** - Built-in health checks

### Production Checklist:
- [ ] Works with `node server.js` immediately
- [ ] Includes all required npm packages
- [ ] Has proper error logging
- [ ] Includes health check endpoint
- [ ] Optimized for server deployment
- [ ] Includes backup/recovery procedures
